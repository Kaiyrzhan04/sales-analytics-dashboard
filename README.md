#  Sales Analytics Dashboard

> End-to-end аналитический проект на основе реального датасета **Online Retail II** (1M+ транзакций, 2009–2011 гг.)

---

## Цель проекта

Построить полный цикл анализа данных: от сырых данных до интерактивного дашборда и прогноза продаж — как это делается в реальных компаниях.

##  Что сделано

### 1. Обработка данных
- Загрузил датасет: **1 067 371 строк**, 8 столбцов
- Удалил дубликаты и отменённые заказы (префикс `C`)
- Создал новые признаки: `Revenue = Quantity × UnitPrice`, сезонные признаки

### 2. RFM-анализ
Сегментировал **5 878 уникальных клиентов** на 8 групп по трём метрикам:
- **Recency** — как давно совершали покупку
- **Frequency** — как часто покупают
- **Monetary** — сколько тратят

| Сегмент | Клиентов | Доля выручки |
|---------|---------|-------------|
| Champions | 11% | **48%** |
| Loyal Customers | 18% | 24% |
| At Risk | 14% | 12% |
| Остальные | 57% | 16% |

>  **Вывод:** 11% лучших клиентов генерируют почти половину всей выручки

### 3. База данных (SQLite)
Спроектировал схему БД и написал аналитические запросы:
- Выручка по месяцам
- Топ-10 товаров по продажам
- Распределение клиентов по сегментам
- Анализ продаж по странам

### 4. Прогнозирование продаж
- Модель: **Linear Regression** с сезонными признаками (ряды Фурье)
- Горизонт прогноза: **3 месяца вперёд**
- Признаки: тренд, месяц, Фурье-компоненты сезонности

### 5. Дашборд Power BI
Интерактивный отчёт включает:
- KPI-карты (выручка, кол-во заказов, средний чек)
- График выручки с наложенным прогнозом
- RFM-сегментацию клиентов
- Карту продаж по странам

---

##  Ключевые выводы

- **Champions (11% клиентов) → 48% выручки** — приоритет для retention-программ
- Пик продаж: **ноябрь–декабрь** (сезонность ~+40% к среднему)
- **Великобритания** — 85% выручки, остальные 37 стран — 15%
- Сегмент **"At Risk"** (14% клиентов) — потенциал реактивации на ~12% выручки
